# Axentis Market — инструкции для Claude

## Обязательное правило: использование скиллов

В этом репозитории установлены скиллы (`.claude/skills/` → `.agents/skills/`). **Всегда автоматически применяй их** при создании нового кода и редактировании существующего — не жди, пока пользователь попросит:

### При любой работе над UI (страницы, компоненты, экраны, вёрстка)
- **ui-ux-pro-max** — сначала подбери стиль, палитру, типографику и UX-гайдлайны через поиск по базе скилла
- **ui-styling** — используй для shadcn/ui, Tailwind, тем, тёмного режима и доступности
- **design-system** — соблюдай токены дизайна (цвета, отступы, типографика), не хардкодь значения
- **apple-design** и **emil-design-eng** — применяй их принципы полировки UI: сдержанность, обратная связь, внимание к деталям

### При любой работе с анимациями и движением
- **motion-dev-animations** — используй Motion.dev (Framer Motion) со spring-физикой; всегда учитывай `prefers-reduced-motion` и производительность ≥60fps
- **improve-animations** / **review-animations** — при аудите и ревью анимаций
- **animation-vocabulary** — для точной терминологии эффектов

### При работе с брендом и графикой
- **brand** — голос бренда, визуальная идентичность, консистентность
- **design** — логотипы, иконки, баннеры, фирменный стиль
- **banner-design** — баннеры для соцсетей, рекламы, hero-секций
- **slides** — презентации

### Порядок работы
1. Перед созданием или изменением UI/анимаций/графики — открой соответствующий SKILL.md и следуй ему
2. Новые экраны и компоненты должны соответствовать единой дизайн-системе проекта
3. Качество по умолчанию: продакшн-уровень полировки, доступность, отзывчивость на всех экранах
