# Axentis Business — мобильная панель компании

React Native (Expo) приложение панели компаний маркетплейса **Axentis**.
Функции 1:1 с веб-панелью компаний, но UI спроектирован заново под телефоны
(Android/iOS): нижняя навигация, bottom-sheets, свайпы, pull-to-refresh,
хаптика, крупные touch-цели.

## Разделы и покрытие функций веб-панели

| Раздел | Функции (1:1 с вебом) |
|---|---|
| **Вход** | телефон + пароль + реферальный код + политика; только `/auth/login/company` |
| **Дашборд** | метрики дня, разбор прибыли онлайн/офлайн (по нажатию), «требует внимания», динамика продаж, статусы заказов, лидеры продаж, прогноз остатков, ABC-анализ, замороженные деньги, сегменты клиентов (VIP/постоянные/новые/засыпающие/потерянные) |
| **Склад** | статистика склада, поиск, фильтр по категориям, «залежавшиеся», карточка товара (все поля), фото (просмотр/загрузка из галереи/удаление), закупка (остаток+цена), варианты SKU (CRUD + закупка варианта), массовый выбор (long-press): в продажу / с продажи / удалить / скидка на модерацию, FAB-добавление |
| **Продажи** | витрина: сетка с фото, выбор товаров, «выставить для покупателей» / «убрать с продажи», выбрать все, статистика; история: офлайн-продажи, итоги, экспорт CSV |
| **Заказы** | поток статусов: Принять → Отправить → Завершить (с частичными возвратами по позициям) / Отменить; фильтры по статусу и периоду (день/неделя/месяц/год), итоги периода (заказы/выручка/наценка), звонок покупателю, адрес доставки, автообновление |
| **Аналитика** | вкладки Финансы/Закупки/Расходы; периоды с дельтой к прошлому периоду; выручка/чистая прибыль (наценка − расходы)/средний чек; онлайн vs офлайн; себестоимость склада; график по дням; топ товаров; разбор по категориям; журнал закупок; учёт расходов (добавить/удалить) |
| **Настройки** | профиль; режим магазина публичный ↔ закрытый (+код доступа с копированием); доставка (радиус/цена за км) и возвраты (окно в часах); регионы обслуживания; Telegram-уведомления (подключить/отключить); язык uz/ru; тема светлая/тёмная; выход |

## ⚡ Wow-функции (только в приложении)

| Функция | Как работает |
|---|---|
| **📷 Касса-сканер** | центральная кнопка таб-бара → камера сканирует штрих-код (EAN/UPC/Code128/QR) → корзина → оплата наличными/картой (UzCard/Humo/Visa) → `/cash-sales`; ручной ввод кода и фонарик; чек можно отправить любым мессенджером |
| **📴 Офлайн-режим кассы** | без интернета продажа сохраняется в локальную очередь (`src/offline.ts`) и досылается автоматически (при старте, при открытии кассы, раз в минуту); каталог товаров кэшируется для офлайн-поиска |
| **🔔 Push о новых заказах** | бэкенд шлёт Expo-push продавцу сразу при оформлении заказа (`NotifyCompanyNewOrder` в `CreateOrder`); токен: `PUT /companies/:id/push-token` |
| **🗣 Утренняя сводка** | воркер `RunCompanyPushWorkers` в 08:00 по Ташкенту: вчерашние заказы/касса/выручка + сколько товаров заканчивается; вкл/выкл в настройках |
| **🤖 Умный план закупки** | кнопка «Что докупить?» на складе: из прогноза остатков (90 дней продаж) считает запас на 14 дней вперёд, количество и бюджет; план шарится поставщику одной кнопкой |

⚠️ Remote push не работают в Expo Go (SDK 53+) — только в dev/production
сборках EAS. Касса и советчик работают везде.

Дизайн-токены (цвета, радиусы, отступы) — 1:1 с `--ax-*` переменными
из `src/dark-theme.css` веб-панели (`src/theme.tsx`).

Не переносилось осознанно (десктоп/веб-специфика): импорт из Excel,
карта доставки, панели SMM/сторис/чат/курьеры/вопросы — по требованию
в приложении только основные разделы.

## 🚫 Безопасность: без админ-панели

В приложении **невозможно войти администратором**:

- используется только `POST /auth/login/company` — каскада
  «админ → компания → агент», как в вебе, здесь нет;
- ни один `/admin/*` endpoint и `/auth/login/admin` в клиент не подключены;
- экран входа явно сообщает, что вход администратора доступен только в веб-панели.

## Запуск в разработке

```bash
cd mobile-company
npm install
npx expo start          # QR-код для Expo Go (Android/iOS)
```

API по умолчанию — `https://axentis.uz/api`. Для локального бэкенда:

```bash
EXPO_PUBLIC_API_URL=http://192.168.x.x:3000/api npx expo start
```

## Сборка для Play Market (EAS)

```bash
npm install -g eas-cli
eas login
eas build -p android --profile preview      # APK для теста на устройстве
eas build -p android --profile production   # AAB для Google Play Console
eas submit -p android                       # загрузка в Play Market
```

Android package: `uz.axentis.business`.
