# Telegram-оповещения для магазинов (5 минут настройки)

## Что это

Бот шлёт магазину в Telegram ТОЛЬКО критичное (всё остальное — на сайте):

1. **⚠️ Критический остаток** — когда остаток товара падает до половины
   порога панели «критических товаров»:
   - дешёвый товар (цена ниже средней по магазину): порог 20 шт → сигнал при **10 шт**;
   - дорогой товар (цена выше средней): порог 10 шт → сигнал при **5 шт**.

   Один сигнал на товар, без спама: после пополнения склада выше полного
   порога флаг сбрасывается, и при новом падении придёт новое оповещение.

2. **📊 Дневной отчёт в 21:00 (Ташкент)** — одно сообщение: заказы онлайн,
   касса (офлайн), наценка (навар), отмены, количество критических товаров.

## Настройка (один раз, владельцем проекта)

1. В Telegram откройте **@BotFather** → `/newbot` → задайте имя
   (например, «Axentis Market Alerts») и username (например,
   `axentis_alerts_bot`). BotFather выдаст **токен** вида
   `1234567890:AAE...`.

2. Добавьте токен в `backend/.env`:

   ```
   TELEGRAM_BOT_TOKEN=1234567890:AAE...
   ```

3. Перезапустите бэкенд (`docker compose up -d --build backend`).
   В логе появится `🤖 Telegram-бот подключён: @axentis_alerts_bot`.

Без токена подсистема полностью выключена и ни на что не влияет.

## Как подключается магазин (сам, без вас)

Панель компании → **Настройки** → карточка «Telegram-оповещения» →
кнопка «Подключить в Telegram» → в открывшемся боте нажать **Start**.
Всё: чат привязан, придёт подтверждение. Отключение — там же.
